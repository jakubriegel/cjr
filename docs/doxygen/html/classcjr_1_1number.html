Universal Values

Values with the same representation with any base.

When using them the library is not considering their base. When base is needed for calculations, it is being taken from second number involved in it. Custom universal values can be created with dedicated method. Since their base is stored as 0, they should be used only when explicitly needed.

Bug:
Undefined behaviour when making calculations only on these values.
